# WB3 탄소저감량 계산방식 분석서

> 작성일: 2026-03-10
> 목적: 중고거래를 통한 탄소저감량 계산 로직이 코드에 어떻게 적용되어 있는지 파악

---

## 1. 요약

라이트브라더스의 탄소저감량 계산은 **3가지 경로**로 이루어져 있다.

| # | 경로 | 계산 방식 | 적용 시스템 |
|---|------|-----------|-------------|
| 1 | **라이딩(자전거 주행)** | 거리(km) × 배출계수 | WB3(Node.js) + AWS2(Java) |
| 2 | **중고거래(상품)** | 프레임 소재 × 무게 기반 | AWS2(Java)만 |
| 3 | **일일 누적 배치** | SP_carbonCalculator 프로시저 | AWS2(Java) 배치 |

---

## 2. 경로별 상세 분석

### 2.1 라이딩 기반 탄소저감량

#### WB3 시스템 (현재 운영 중)

**파일:** `API/api/routes/v1/user/mypage/riding.js` (109번째 줄)

```javascript
params.carbonReduce = distance ? distance * 0.2375 : 0;
```

- **계산식:** `탄소저감량(kg) = 라이딩 거리(km) × 0.2375`
- **고정 계수:** 0.2375 kg/km (연료·차종 구분 없이 단일 값)
- **저장 위치:** `T_RIDING` 테이블의 `CARBON_REDUCE` 컬럼
- **트리거 시점:** 사용자가 라이딩 후 설문(survey) 제출 시

#### AWS2 시스템 (리뉴얼 버전)

**파일:** `AWS2/wb-platform-server-master/.../riding/service/RidingService.java` (216~301번째 줄)

AWS2에서는 **연료 종류(F)와 차종(C)에 따라 다른 배출계수**를 적용하는 고급 계산 로직을 사용한다.

```
탄소저감량(kg) = 대체이동수단 거리(km) × 연료·차종별 배출계수
```

**연료·차종별 배출계수 표:**

| 연료 코드 | 연료 | 배출계수 (kg CO₂/km) |
|-----------|------|---------------------|
| F01 | 휘발유 | 0.15871 |
| F02 | 경유 | 0.19701 |
| F03 | LPG | 0.00182 |
| F04 | 하이브리드 | 0.1255 |
| F05 | 전기 | 0.1255 (경차) / 0.0094 (그 외) |
| F99 | 기타 (미선택) | 0.0094 |

> 특이사항: F01(휘발유)과 F05(전기)에서 대체수단이 **이륜차(R09)**인 경우 별도 계수 적용
> - 휘발유+이륜: 0.0094
> - 전기+이륜: 0.0791

**저장 위치:** `exch_actvs` 테이블의 `carbon_rdu` 컬럼

### 2.2 중고거래(상품) 기반 탄소저감량

**파일:** `AWS2/wb-platform-server-master/.../product/repository/ProductCarbonRepository.java`

**이 기능은 AWS2 시스템에만 존재**하며, WB3에는 구현되어 있지 않다.

자전거 **프레임 소재와 무게**를 기반으로 중고거래 시 저감되는 탄소량을 계산한다.

```
탄소저감량 = 프레임소재별 계수 × 무게(kg)
```

**프레임 소재별 계산 방식:**

| 소재 코드 | 소재 | 무게 있을 때 | 무게 없을 때 (기본값) |
|-----------|------|-------------|---------------------|
| F01 | 알루미늄 | 1.52 × 무게 | 15.143 kg |
| F02 | 크로몰리(철) | 7.875 × 무게 | 88.765 kg |
| F03 | 카본 | 26.07 × 무게 | 209.648 kg |
| F04 | 티타늄 | 8.895 × 무게 | 56.394 kg |
| F05 | 혼합/기타 | 12.42375 × 무게 | 225.263 kg |
| (미입력) | - | - | 185.654 kg |

> **비유로 설명하면:** 새 자전거를 제조할 때 발생하는 탄소를 중고거래로 "안 만들어도 되니까" 그만큼 줄인 것으로 본다.
> 카본 프레임이 가장 계수가 높은 이유는, 카본 제조 과정이 가장 에너지 집약적이기 때문이다.

**호출 위치:** `ProductService.java` (337번째 줄)에서 상품 상세 조회 시 호출

### 2.3 일일 누적 배치 (탄소계산기)

**파일들:**
- `AWS2/wb-admin-server-master/.../carbon/CarbonCalculatorJob.java` — 배치 Job
- `AWS2/wb-admin-server-master/.../carbon/service/CarbonCalculatorService.java` — 서비스
- `AWS2/wb-admin-server-master/.../carbon/query/CarbonCalculator.xml` — SQL 매퍼

매일 배치로 `SP_carbonCalculator()` 저장 프로시저를 실행하여 **전체 누적 탄소저감량**을 계산한다.

**결과 저장 테이블:** `crbn_calc`

| 컬럼 | 의미 | 계산식 |
|------|------|--------|
| crbn | 누적 탄소저감량 (kg) | 중고거래 + 라이딩 합산 |
| dstn | 누적 라이딩 거리 (km) | - |
| earth | 지구 바퀴 수 | 거리 / 40,120 km |
| car_cnt | 자동차 대수 환산 | 라이딩 탄소저감량 / 3.61 |
| ride_crbn | 라이딩 탄소저감량 (kg) | 거리 × 0.2375 |
| tree_crbn | 소나무 그루 수 환산 | 라이딩 탄소저감량 / 10.77 |
| air_crbn | 에어컨 가동시간 환산 | 총 탄소저감량 / 0.9428 |
| cfe_crbn | 커피 잔 수 환산 | 총 탄소저감량 / (15.3 / 142) |
| snack_crbn | 과자 봉지 수 환산 | 총 탄소저감량 / 0.25 |

> 주의: 실제 계산 로직은 DB 저장 프로시저(`SP_carbonCalculator`) 안에 있어 코드만으로는 내부 로직을 확인할 수 없다.

## 3. 프론트엔드 표시

### 탄소리포트 페이지

**파일:** `AWS2/wb-platform-client-master/.../mypage/carbon/carbonReport.vue`

- API: `GET /mypage/carbonReport` → `crbn_calc` 테이블 최신 레코드 조회
- "모두의 중고거래와 라이딩이 모여서 탄소 저감을 하고있어요" 문구 표시
- 누적 탄소량, 소나무 그루 수, 에어컨 가동시간, 커피 잔 수, 과자 봉지 수 등으로 환산하여 표시

### 지속가능성 페이지

**파일:** `AWS2/wb-platform-client-master/.../guide/SustainabilityWrap.vue`

- API: `GET /carbon-emission/today` → 오늘자 누적 탄소저감 메트릭 조회
- 비행기 탄소배출량, 나무, 자동차로 환산 비교 UI 제공
- "중고거래를 통한 자원순환과 라이딩으로 얼마만큼의 탄소를 저감했을까요?" 문구 표시

### WB3 앱 마이페이지

**파일:** `WEB/src/web/Components/App/Mypage/Riding.js` (425번째 줄)

- 개별 라이딩 기록마다 `carbonReduce` 값을 Kg 단위로 표시

## 4. WB3 vs AWS2 비교

| 항목 | WB3 (현재 운영) | AWS2 (리뉴얼) |
|------|----------------|---------------|
| **라이딩 탄소저감** | 단일 계수 0.2375 | 연료·차종별 세분화 (5×5 매트릭스) |
| **중고거래 탄소저감** | 없음 | 프레임 소재·무게 기반 계산 |
| **누적 배치** | `T_STATS_PLATFORM.RIDING_SUM_CARBON`으로 일별 집계 | `SP_carbonCalculator` 프로시저로 상세 집계 |
| **공항간 탄소배출 비교** | 없음 | `inter_airports_carbon_emission` 테이블 |
| **탄소리포트 페이지** | 없음 | 전용 페이지 (소나무·에어컨·커피·과자 환산) |
| **지속가능성 페이지** | 없음 | 비행기·나무·자동차 비교 UI |

## 5. 현재 상태 판단

### 아직 적용 중인 것 (WB3)

1. **라이딩 탄소저감 계산** — `distance × 0.2375` 공식이 `API/api/routes/v1/user/mypage/riding.js`에서 활성 상태
2. **일별 통계 집계** — `API/mapper/cron_stats.xml`에서 `SUM(CARBON_REDUCE)`로 `T_STATS_PLATFORM`에 기록

### 코드는 있지만 WB3에서 직접 사용하지 않는 것

1. **중고거래(상품) 탄소저감** — AWS2의 `ProductCarbonRepository`에만 존재. WB3에는 해당 로직 없음
2. **연료·차종별 세분화 계산** — AWS2의 `RidingService.carbonCalculate()`에만 존재
3. **탄소계산기 배치** — AWS2의 `CarbonCalculatorJob`에서 실행. WB3는 별도의 크론 통계만 사용
4. **공항간 탄소배출 비교 기능** — AWS2 전용 기능

---

## 6. 결론

> **중고거래를 통한 탄소저감량 계산은 AWS2 시스템에만 구현되어 있으며, 현재 운영 중인 WB3에는 적용되어 있지 않다.**

WB3에서 작동 중인 탄소저감 기능은 **라이딩(자전거 주행) 기반**의 단순 계산(`거리 × 0.2375`)뿐이다. 중고거래 시 자전거 프레임 소재·무게를 기반으로 탄소저감량을 산출하는 로직은 AWS2의 `ProductCarbonRepository.searchproductCarbon()` 메서드에 구현되어 있지만, WB3 Node.js API에는 대응 코드가 없다.

다만, WB3의 일별 플랫폼 통계(`T_STATS_PLATFORM`)에서는 라이딩의 `CARBON_REDUCE` 합계만 집계하고 있어, 중고거래 탄소저감량은 WB3 통계에도 포함되지 않는다.
