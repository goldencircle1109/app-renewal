> 작성일: 2026-03-19
> 목적: 특정매입 + 익명 판매 구조의 법적/세무/회계 리스크 점검

## 1. 핵심 결론 요약

| 항목 | 판정 | 요약 |
|------|------|------|
| 사업 모델 적법성 | ✅ **적법** | 특정매입 구조에서 딜러 비공개는 법적 의무 위반 아님 |
| 수입사 가격 강제 | ⚠️ **수입사가 위법** | 재판매가격유지행위는 공정거래법 위반 |
| 대리점 리스크 | ⚠️ **계약 관리 필요** | 딜러의 대리점 계약 위반 책임은 딜러에게 |
| 회계 처리 | ⚠️ **구조 설계 중요** | 총액법 적용 위해 직매입 요건 충족 필수 |
| 세무 리스크 | ⚠️ **관리 필요** | 부가세/법인세 시점 차이 관리 필수 |
| 소비자 보호 | ✅ **문제 없음** | 라브가 법적 판매자이므로 딜러 공개 의무 없음 |

## 2. 공정거래법 분석

### 2-1. 재판매가격유지행위(RPM)란?

수입사/제조사가 대리점에게 **"이 가격 이하로 팔지 마라"** 고 강제하는 행위입니다.

```
공정거래법 제46조 (재판매가격유지행위의 제한)

"사업자는 재판매가격유지행위를 하여서는 아니 된다."

→ 최저재판매가격 유지행위는 원칙적으로 위법!
```

### 2-2. 이것은 수입사가 위법한 것이다

**결정적 판례: 던롭스포츠코리아 사건 (2025년 3월)**

- 공정위가 던롭스포츠코리아(젝시오 골프채)에 **과징금 18억 6,500만원** 부과
- 스포츠 용품 분야에서 제조사의 가격 강제가 **명백히 위법**임이 확인됨
- 공정위 의결: "온·오프라인 소매가격을 감시하고, 할인 판매한 유통업자에게 불이익을 주는 행위"

**피아노 제조사 사건 (2024년, 의결 2024-074)**

- 온라인 MAP(Minimum Advertised Price) 강제도 위법 판단
- "온라인에서 권장가보다 낮은 가격을 게시하지 못하게 하는 행위" = 재판매가격유지행위

### 2-3. 라이트브라더스의 법적 입장

```
[라브는 안전한 쪽에 있다]

라브는 "가격을 낮추는 행위자"이므로:
- 재판매가격유지행위의 가해자가 아님 ✅
- 오히려 소비자 이익 증진에 기여 ✅
- 공정거래법 직접 위반 위험: 매우 낮음 ✅

수입사가 라브를 공격할 수 있는 방법:
- "우리 브랜드를 할인 판매하지 마라" → 법적 근거 약함
- 대리점에 "라브에 납품하면 계약 해지" → 이것도 RPM의 연장이라 위법 소지

하지만 현실적으로:
- 수입사가 공정위에 신고당할 위험을 무릅쓰고 라브를 공격할 가능성 낮음
- 특히 던롭 사건 이후 업계가 매우 조심스러워진 상황
```

### 2-4. 다만, 대리점 계약 문제

대리점이 수입사와 맺은 계약서에 아래 조항이 있을 수 있습니다:
- "제3자에게 상품을 공급할 때 사전 승인 필요"
- "지정 채널 이외의 유통 금지"

이런 조항 자체의 유효성은 공정거래법상 다툼의 여지가 있지만, **현실적으로 대리점이 리스크를 부담**하는 것은 사실입니다.

**대응 방안:**
- 딜러 계약서에 **"대리점 계약 위반으로 인한 모든 책임은 딜러 본인에게 있음"** 명시
- 딜러에게 참여 전 본인의 대리점 계약서를 확인하도록 안내
- NDA(비밀유지계약) 체결 → 딜러 정보가 외부로 유출되지 않도록 보호

## 3. 특정매입 법적 구조 분석

### 3-1. 왜 특정매입인가?

| 거래 형태 | 법적 판매자 | 딜러 노출 | 매출 인식 |
|----------|-----------|----------|----------|
| **직매입** | 라브 | ❌ 비공개 | 전액 (총액법) |
| **특정매입(특약매입)** | 라브 | ❌ 비공개 | 전액 또는 수수료 |
| 위탁매입 | **딜러** | ⚠️ 공개 필요 | 수수료만 |
| 오픈마켓(중개) | **딜러** | ⚠️ 공개 필요 | 수수료만 |

> **위탁매입/오픈마켓 구조는 딜러 정보 공개 의무가 발생**하므로
> 깜깜이방의 핵심인 "익명성"을 보장할 수 없습니다.
> **반드시 직매입 또는 특정매입 구조**를 사용해야 합니다.

### 3-2. 직매입 구조를 권장하는 이유

사실 깜깜이방의 실질에 가장 부합하는 것은 **직매입(확정 매입)** 입니다.

```
직매입 구조:

1. 딜러가 상품을 라브에 "판매" (소유권 완전 이전)
2. 라브가 소비자에게 "재판매" (자기 상품을 파는 것)
3. 반품: 소비자 → 라브 (딜러 관련 없음)

장점:
- 법적으로 가장 명확
- 회계/세무 처리 단순
- 소비자보호법상 라브가 판매자 → 딜러 공개 의무 제로
- 감사인 질의 위험 최소화

단점:
- 재고 위험을 라브가 전부 부담
- 매입 자금이 필요 (선결제)
```

### 3-3. 실질적 해결책: "조건부 직매입"

완전 직매입은 자금 부담이 크므로, 실무적으로는:

```
[조건부 직매입 구조]

1. 딜러가 상품 등록 (사진 + 정보)
2. 소비자가 주문하면 그 시점에 라브가 딜러로부터 "매입"
3. 소유권이 딜러 → 라브로 이전
4. 라브가 소비자에게 "판매"

→ 사전 재고 부담 없이 직매입 형태를 유지
→ 법적으로는 라브가 매입 후 재판매한 것
→ 계약서에 "주문 시점에 소유권 이전" 조항 명시
```

> 이 구조가 **백화점의 특정매입(특약매입)** 과 거의 동일합니다.
> 백화점도 실물을 창고에 보관하지 않고, 매장(대리점)에서 직접 출고합니다.

## 4. 전자상거래법(소비자보호) 분석

### 4-1. 판매자 정보 공시 의무

```
전자상거래법 제13조 (신원 및 거래조건에 대한 정보의 제공)

통신판매업자는 다음 사항을 소비자에게 제공해야 한다:
1. 상호 및 대표자 성명
2. 주소, 전화번호
3. 사업자등록번호
...
```

**핵심**: 여기서 "통신판매업자" = **법적 판매자**

- 직매입/특정매입 → 법적 판매자 = 라이트브라더스 → **라브 정보만 공시하면 됨** ✅
- 위탁매입 → 법적 판매자 = 딜러 → 딜러 정보 공시 필요 ❌

### 4-2. 제조물책임

```
제조물책임법:
- 제조자 = 1차 책임
- 판매자 = 제조자를 알 수 없거나 연락 불가 시 보충적 책임

라브가 직매입으로 "판매자"이므로:
→ 제조물 결함 시 라브에게 1차 클레임 올 수 있음
→ 라브는 딜러에게 구상권 행사 가능

대응: 딜러 계약서에 "제조물 결함에 대한 구상권 보장" 조항 필수
```

### 4-3. 반품/환불

```
소비자 반품 흐름:

소비자 → 라이트브라더스 (반품 접수)
    → 라브 주소로 반송 (또는 라브 지정 주소)
    → 라브가 환불 처리
    → 라브가 딜러에게 반품 전달 및 매입가 환수

핵심: 소비자는 라브하고만 소통, 딜러 존재를 모름
```

### 4-4. A/S 정책

```
⚠️ 주의: A/S 정보 불명확 시 기만적 표시 위험

자전거는 A/S가 특히 중요한 상품!

대응:
1. 상품 페이지에 "A/S 문의: 라이트브라더스 고객센터" 명시
2. 라브가 1차 접수 → 제조사 서비스센터 연결 또는 직접 처리
3. 보증 기간/범위 명확히 표시

→ "판매자가 A/S를 안 해준다"는 민원이 나오면 안 됨
```

## 5. 회계 처리 분석

### 5-1. 총액법 vs 순액법: 무엇이 다른가?

같은 거래(매입가 80만원, 판매가 100만원)를 회계 처리할 때:

```
[총액법 - 라브가 "본인(Principal)"인 경우]

매출액:     1,000,000원  (판매가 전액)
매출원가:     800,000원  (매입가)
매출총이익:   200,000원

→ 매출이 크게 잡힘 → 외형 성장에 유리
→ 투자 유치, 사업 확장 시 유리
```

```
[순액법 - 라브가 "대리인(Agent)"인 경우]

수수료수익:  200,000원  (마진만)
매출원가:          0원

→ 매출이 작게 잡힘 → 이익률은 높아 보임
→ 실질 이익은 동일하지만 외형이 작음
```

### 5-2. 총액법 적용 요건 (K-IFRS 1115호 B34~B38)

총액법을 적용하려면 라브가 **"본인(Principal)"** 이어야 합니다.

```
[본인 판정 3대 기준]

① 주된 책임 (Primary Responsibility)
   "상품 제공에 대한 주된 책임이 라브에게 있는가?"
   → 라브 이름으로 판매, A/S도 라브가 담당 → ✅ 충족

② 재고 위험 (Inventory Risk)
   "팔리지 않았을 때의 손실을 라브가 부담하는가?"
   → 직매입 구조: 매입 후 반품 불가 → ✅ 충족
   → 특정매입(반품 가능): ⚠️ 미충족 가능성

③ 가격결정권 (Pricing Discretion)
   "라브가 판매 가격을 자유롭게 정할 수 있는가?"
   → 딜러 납품가 이상에서 자유 결정 → ✅ 충족
```

### 5-3. 총액법 적용을 위한 계약서 필수 조항

```
[계약서에 반드시 포함해야 할 5개 조항]

1. 소유권 이전:
   "주문 접수 시점에 상품의 소유권은 딜러에서 라브로 완전히 이전된다"

2. 반품 제한:
   "라브는 매입한 상품에 대해 딜러에게 반품을 요구할 수 없다"
   (또는 반품은 상품 하자 시에만 가능하도록 엄격 제한)

3. 가격결정:
   "라브는 소비자 판매가를 독자적으로 결정할 권리를 가진다"

4. 위험 이전:
   "상품 멸실, 훼손, 감가에 대한 위험은 매입 시점부터 라브가 부담한다"

5. 클레임 처리:
   "소비자 반품, 교환, A/S는 라브가 직접 처리한다"
```

### 5-4. 회계 처리 예시 (직매입/총액법)

```
[1단계: 소비자 주문 시 (= 딜러로부터 매입)]

(차) 상품(재고자산)   800,000
(대) 매입채무(딜러)   800,000

[2단계: 소비자에게 판매/배송 완료 시]

(차) 매출채권/현금   1,000,000
(대) 매출액         1,000,000

(차) 매출원가        800,000
(대) 상품(재고자산)  800,000

[3단계: 딜러 정산]

(차) 매입채무(딜러)   800,000
(대) 보통예금         800,000
```

### 5-5. 백화점 사례가 주는 교훈

```
2004년 회계기준 변경 사건:

변경 전: 백화점들이 특정매입을 총액법으로 처리
→ 롯데백화점 매출 수조원

변경 후: 금감원이 "순액법으로 하라" 지시
→ 롯데백화점 매출이 수천억원으로 축소
→ 신세계가 22년 만에 업계 1위 탈환

교훈: 같은 실적이라도 회계 처리 방식에 따라 매출 순위가 바뀜

라브 시사점:
→ 총액법을 쓰려면 "진짜 직매입"이어야 함
→ 실질 없이 형식만 직매입이면 감사인/국세청에 적발될 수 있음
→ 계약서 + 실제 운영 둘 다 직매입 요건을 충족해야 함
```

## 6. 세금 처리 분석

### 6-1. 부가가치세(VAT) 흐름

```
[직매입 구조 - 세금계산서 흐름]

딜러 ──(세금계산서)──→ 라이트브라더스 ──(세금계산서/영수증)──→ 소비자

딜러 발행:
- 공급자: 딜러 (사업자)
- 공급받는자: 라이트브라더스
- 공급가액: 매입가 (예: 800,000원)
- 부가세: 80,000원

라브 발행 (B2C - 개인 소비자):
- 신용카드 매출전표 또는 현금영수증
- 금액: 1,000,000원 + VAT 100,000원

라브 발행 (B2B - 법인 소비자):
- 세금계산서
- 공급가액: 1,000,000원
- 부가세: 100,000원
```

### 6-2. 부가세 납부/공제

```
라브의 부가세 계산:

매출세액: 100,000원 (소비자 판매 시)
매입세액:  80,000원 (딜러 매입 시)
─────────────────────
납부세액:  20,000원 (= 마진에 대한 부가세)
```

### 6-3. 법인세

```
라브의 법인세 과세소득:

매출:       1,000,000원
매출원가:     800,000원
매출총이익:   200,000원
→ 이 200,000원에 대해 법인세 과세
```

### 6-4. ⚠️ 세무 리스크: 공급시기 불일치

```
[주의 사항]

부가가치세법: 공급시기 = 상품 인도일 (딜러 → 라브 인도 시)
법인세법:     수익인식 = 소비자에게 판매 완료 시

→ 딜러가 라브에 납품한 날과 소비자에게 판매된 날이 다를 수 있음
→ 이 시차를 제대로 관리하지 않으면 세무조사 시 문제

[관련 판례]
조세심판원 조심2023서7120:
- 납품 시점과 판매 시점의 데이터 불일치로 대규모 세무 분쟁 발생
- ERP/회계 시스템에서 두 시점을 모두 정확히 기록해야 함
```

**대응 방안:**
1. 깜깜이방은 **주문 즉시 매입** 구조이므로 시차가 최소화됨
2. 그래도 매입일, 출고일, 배송완료일을 모두 기록하는 시스템 필수
3. 분기별 세무사 검토 권장

### 7-2. 가장 중요한 리스크 대응 3가지

```
① 익명성 유출 방지 (사업 존망과 직결)
   - 딜러 정보 접근 권한: 대표 + 운영담당 2명만
   - 배송 송장: 반드시 라브 명의
   - CS 응대: 절대 딜러 정보 언급 금지
   - DB 접근: DARK_PARTNER_ID 컬럼 암호화
   - 딜러와의 소통: 전용 채널 (일반 CS와 분리)

② 세무 리스크 관리 (장기 지속성)
   - 계약서: 직매입 요건 5개 조항 모두 포함
   - 회계: 매입일/판매일 정확히 기록
   - 세금계산서: 발행 타임라인 엄수
   - 정기 검토: 분기 1회 세무사 리뷰

③ 품질 관리 (브랜드 보호)
   - 딜러 등록 시 사업자등록증 + 대리점 자격 확인
   - 상품 등록 시 사진 3장 이상 필수
   - 상태 허위 기재 시 패널티 (수수료 인상 → 퇴출)
   - 소비자 불만 2건 이상 시 해당 딜러 일시 정지
```
