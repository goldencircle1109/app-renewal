# 깜깜이방 (Dark Commerce) 사업기획서

> 작성일: 2026-03-19
> 프로젝트: 라이트브라더스 - 익명 할인 마켓플레이스

## 1. 사업 개요

### 1-1. 문제 정의

자전거, 부품, 러닝화 등 스포츠 용품 업계에는 **수입사/본사의 가격정책**이 존재합니다.

```
[현재 구조]

본사/수입사 ──(권장소비자가격 강제)──→ 대리점
                                        │
                                        ├─ 할인 판매 → 대리점 계약 해지 위협
                                        ├─ 재고 쌓임 → 현금 유동성 악화
                                        └─ 결국 폐업 또는 원가 이하 떨이
```

**대리점의 고통:**
- 시즌 지난 재고를 할인하고 싶지만 **공개적으로 할인하면** 수입사에서 계약 해지
- 매장에 쌓인 재고가 현금을 묶어놓아 **새 시즌 상품 매입이 불가능**
- 온라인 가격 모니터링이 강화되어 **자기 이름으로는 절대 할인 불가**

### 1-2. 해결 방안: 깜깜이방

```
[깜깜이방 구조]

대리점 A ─┐
대리점 B ─┤                           소비자
대리점 C ─┼──(특정매입)──→ 라이트브라더스 ──(판매)──→ (누가 파는지 모름)
대리점 D ─┤               (라브 이름으로 판매)
대리점 E ─┘
```

- **대리점**: 할인 상품을 라브에 납품 → 누가 팔았는지 **아무도 모름** → 현금 확보
- **라이트브라더스**: 수수료 수익 + **전부 상품매출로 잡힘** (특정매입이라 거래량이 아닌 매출)
- **소비자**: 정품을 할인가에 구매 (판매자는 라이트브라더스로만 표시)

### 1-3. 왜 "깜깜이방"인가?

| 구분 | 일반 마켓플레이스 | 깜깜이방 |
|------|-----------------|---------|
| 판매자 표시 | 대리점 이름 공개 | **라이트브라더스만 표시** |
| 수입사 추적 | 어떤 대리점이 할인하는지 바로 노출 | **추적 불가** |
| 매출 구조 | 수수료만 매출 | **판매가 전액이 매출** |
| 대리점 리스크 | 수입사 보복 위험 높음 | **거의 없음** |

## 2. 비즈니스 모델

### 2-1. 거래 구조: 특정매입(特定買入)

```
[거래 흐름]

① 대리점이 상품 등록 (할인가 제시)
   예: 로드바이크 정가 300만원 → 대리점 납품가 210만원 (30% 할인)

② 라이트브라더스가 매입 결정
   예: 210만원에 매입 확정

③ 라이트브라더스 이름으로 판매
   예: 240만원에 소비자에게 판매 (정가 대비 20% 할인)

④ 정산
   - 소비자 결제: 240만원
   - 대리점 지급: 210만원
   - 라브 마진: 30만원 (수수료율 12.5%)
```

**특정매입이 핵심인 이유:**
- 라브가 "매입"하여 "재판매"하는 구조이므로 **법적 판매자가 라브**
- 소비자에게 대리점 정보를 공개할 의무가 없음
- 매출액이 수수료가 아닌 **판매가 전액**으로 잡힘

### 2-2. 수수료 구조

| 카테고리 | 수수료율 | 예시 |
|---------|---------|------|
| 완성차 (200만원 이상) | 8~10% | 300만→270만 판매, 마진 24~27만 |
| 완성차 (200만원 미만) | 12~15% | 100만→85만 판매, 마진 10~13만 |
| 고급 부품/파츠 | 10~12% | 50만→44만 판매, 마진 5~6만 |
| 러닝화 | 12~15% | 20만→17만 판매, 마진 2~3만 |
| 의류/헬멧/액세서리 | 15~20% | 10만→8만 판매, 마진 1.5~2만 |

> 참고: 네이버 스마트스토어 2~5%, 쿠팡 5~10%보다 높지만,
> 대리점이 얻는 가치(신원보호 + 재고현금화 + 마케팅 불필요)를 감안하면 합리적

### 2-3. 수익 시뮬레이션

```
[월간 목표 시나리오]

가정:
- 참여 대리점: 30개
- 대리점당 월 등록 상품: 5개
- 평균 판매가: 50만원
- 판매 전환율: 70%
- 평균 수수료율: 12%

계산:
- 월 등록 상품: 30 × 5 = 150개
- 월 판매 상품: 150 × 70% = 105개
- 월 GMV(총거래액): 105 × 50만 = 5,250만원
- 월 매출액(라브): 5,250만원 (특정매입 → 전액 매출)
- 월 수수료 이익: 5,250만 × 12% = 630만원
- 연간 예상 이익: 약 7,560만원
```

### 2-4. 손익분기점

```
월 고정비 추정:
- 추가 인건비 (CS/운영 1명): 300만원
- 서버/인프라 추가 비용: 30만원
- 마케팅비: 100만원
- 기타 (배송/포장 보조): 50만원
= 월 고정비 합계: 480만원

손익분기 GMV = 480만 ÷ 12% = 4,000만원/월
= 평균 50만원 상품 기준 월 80건 판매
= 약 22건/일 (주문 기준)
```

```

## 4. 운영 모델

### 4-1. 상품 등록 ~ 판매 ~ 정산 프로세스

```
[전체 프로세스]

① 대리점 상품 등록 (딜러 전용 페이지)
   ├─ 상품 사진 업로드 (3장 이상)
   ├─ 상품 상태 선택 (새상품/전시품/리퍼)
   ├─ 정가 입력
   ├─ 희망 납품가 입력 (= 대리점이 받고 싶은 금액)
   └─ 판매 기한 설정 (기본 14일)

② 라이트브라더스 검수
   ├─ 상품 정보 확인 (정품 여부, 상태 확인)
   ├─ 판매가 결정 (납품가 + 수수료 + 배송비 고려)
   └─ 승인/거절

③ 판매 개시
   ├─ 앱/웹에 "라이트브라더스" 이름으로 노출
   ├─ 판매 기한까지 Flash Sale 형태 운영
   └─ 재고 1개(원칙) → 선착순 구매

④ 주문 접수
   ├─ 소비자 결제 → 라브 계좌 입금
   ├─ 대리점에 출고 요청 (배송지만 전달, 소비자 정보는 마스킹)
   └─ 대리점이 라브 포장재로 발송 (또는 라브 물류센터 경유)

⑤ 배송 완료 + 구매 확정
   └─ 구매확정 후 3영업일 이내 대리점 정산

⑥ CS/반품
   ├─ 소비자 → 라이트브라더스로 문의 (대리점 연결 X)
   ├─ 반품 시 라브 주소로 반송 → 라브가 대리점에 전달
   └─ 대리점 주소 절대 노출 안 됨
```

### 4-2. 배송 방식 선택

| 방식 | 장점 | 단점 | 권장 |
|------|------|------|------|
| **A. 대리점 직배송** | 라브 물류 비용 없음 | 포장 품질 편차, 대리점 주소 노출 위험 | 단, 라브 송장 사용 |
| **B. 라브 물류센터 경유** | 품질 일관성, 주소 완전 차단 | 배송 시간 추가, 물류 비용 발생 | 고가 상품 |
| **C. 하이브리드** | 상품별 최적화 | 운영 복잡 | **Phase 2부터** |

**Phase 1 권장: A 방식 (대리점 직배송 + 라브 송장)**
- 라브가 택배사와 계약 → 라브 명의 송장 발급
- 대리점은 라브 송장을 붙여서 발송
- 소비자에게는 "라이트브라더스"에서 온 택배로 보임

### 4-3. 판매 형태: Flash Sale

일반 상시 판매가 아닌 **Flash Sale(한정 시간/수량 판매)** 형태를 권장합니다.

```
[Flash Sale 장점]

1. 긴급성 → 구매 전환율 상승
   "이 가격은 48시간만 유효합니다"

2. 수입사 모니터링 회피
   상시 할인이 아니라 짧은 기간만 노출 → 추적 어려움

3. 재고 1개 원칙
   대리점이 가진 실물 1개만 판매 → 재고 위험 최소화

4. 마케팅 효과
   "오늘의 딜", "한정 수량" → 앱 접속 유도
```

## 5. 기술 구현 방안

### 5-1. 기존 WB3 시스템 활용

기존 시스템을 최대한 재사용하여 개발 비용을 최소화합니다.

| 기존 시스템 | 재사용 방법 |
|-----------|-----------|
| T_PARTNER (파트너 테이블) | 딜러를 파트너로 등록 |
| T_GOODS (상품 테이블) | 깜깜이방 상품도 동일 구조 사용 |
| T_GOODS_CONFIRM (검수) | 딜러 상품 승인 프로세스에 활용 |
| T_ORDER (주문) | 주문/결제 프로세스 그대로 사용 |
| 파트너 정산 시스템 | 딜러 정산에 활용 |
| SSP 포인트 시스템 | 깜깜이방 구매 시 포인트 적립 |

### 5-2. 신규 개발 필요 항목

```
[DB 변경 - 최소한]
T_GOODS에 컬럼 추가:
  - IS_DARK_COMMERCE   TINYINT(1)  -- 깜깜이방 상품 여부
  - DARK_EXPIRE_DATE   DATETIME    -- Flash Sale 만료 시간
  - DARK_MIN_PRICE     DECIMAL     -- 대리점 납품가 (최소 정산가)
  - DARK_PARTNER_ID    INT         -- 납품 대리점 (내부 참조용, 외부 비공개)

T_DARK_SETTLEMENT (정산 테이블) 신규:
  - SETTLEMENT_ID      INT PK
  - PARTNER_ID         INT         -- 대리점
  - ORDER_ID           INT         -- 주문
  - GOODS_ID           INT         -- 상품
  - SALE_PRICE         DECIMAL     -- 판매가
  - PARTNER_PRICE      DECIMAL     -- 대리점 납품가
  - COMMISSION         DECIMAL     -- 수수료
  - COMMISSION_RATE    DECIMAL     -- 수수료율
  - STATUS             VARCHAR(20) -- PENDING/CONFIRMED/PAID
  - PAID_AT            DATETIME    -- 정산일
```

```
[신규 페이지]
1. 딜러 전용 - 상품 등록 페이지 (간소화된 폼)
2. 딜러 전용 - 정산 현황 페이지
3. 소비자 - 깜깜이방 메인 (Flash Sale 리스트)
4. 소비자 - 깜깜이방 상품 상세 (타이머 포함)
5. 관리자 - 깜깜이방 상품 검수/승인 페이지
6. 관리자 - 대리점 정산 관리 페이지
```

## 6. 사업 활성화 전략

### 6-1. 콜드스타트 해결: 딜러 확보가 먼저

일반 마켓플레이스와 달리 **딜러의 재고 소진 필요성이 이미 존재**합니다.
소비자가 없어도 딜러는 참여를 고려할 유인이 있습니다.

```
[딜러 영업 핵심 메시지]

"사장님 가게 이름은 절대 나오지 않습니다.
 수수료만 내시고, 재고를 현금으로 바꿔가세요.
 수입사는 모릅니다."
```

**초기 딜러 확보 전략:**

| 순서 | 전략 | 내용 |
|------|------|------|
| 1 | 기존 파트너 활용 | WB와 이미 거래 중인 대리점 3~5곳에 제안 |
| 2 | 수수료 인센티브 | 첫 3개월 수수료 50% 할인 |
| 3 | 빠른 정산 | 구매확정 후 3영업일 이내 입금 (업계 평균 2~4주) |
| 4 | 1:1 방문 영업 | 자전거 밀집 지역 (서울 뚝섬, 양재, 분당) 직접 방문 |
| 5 | 시즌 공략 | 겨울(비시즌) 재고 부담 극대화 시점에 제안 |

### 6-2. 소비자 유입 전략

```
[절대 금지 사항]
- SNS 공개 게시물에 "브랜드명 + 할인율" 병기 금지
  → 수입사 모니터링에 걸림
- 예: "XX브랜드 자전거 30% 할인!" ← 이렇게 하면 안 됨

[허용되는 마케팅]

## 7. 핵심 성공 요인 (KSF)

### 7-1. 절대적으로 지켜야 할 3가지

```
① 익명성 사수
   - 대리점 정보가 어떤 경로로든 노출되면 사업 모델 자체가 무너짐
   - 배송 송장, CS 응대, 반품 주소 모두 라브 이름
   - 내부적으로도 접근 권한 최소화 (관리자 2명만 대리점 매칭 가능)

② 빠른 정산
   - 대리점이 깜깜이방을 쓰는 가장 큰 이유는 "재고를 빨리 현금화"
   - 정산이 늦으면 대리점이 떠남
   - 구매확정 후 3영업일 이내 입금이 최소 기준

③ 품질 관리
   - "라이트브라더스"로 팔리므로 불량 상품은 라브 브랜드에 타격
   - 등록 시 사진 검수 필수
   - 상태 허위 기재 시 딜러 패널티 (수수료 인상 또는 퇴출)
```

### 7-2. 차별화 포인트

| 기존 채널 | 깜깜이방 |
|---------|---------|
| 중고나라/당근마켓 | 정품 보증 없음 | **라브가 정품 보증** |
| 바이크마켓 | 대리점 이름 노출 | **완전 익명** |
| 자체 할인 판매 | 수입사 보복 위험 | **제로 리스크** |
| 떨이 판매 (오프라인) | 매장 방문 고객만 | **전국 온라인 판매** |

## 9. 확장 가능성

### 9-1. 카테고리 확장

```
Phase 1: 자전거 + 러닝화 (현재)
Phase 2: + 골프 (수입사 가격 통제 심함)
Phase 3: + 캠핑 장비 (시즌성 재고 부담)
Phase 4: + 아웃도어 의류/장비
Phase 5: + 전자기기 (카메라, 오디오 등)
```

> **가격 통제가 심한 업종일수록** 깜깜이방의 가치가 높아짐

### 9-2. 서비스 확장

```
[중기]
- 대리점 간 재고 교환 중개 (A매장 잘 팔리는 상품을 B매장에서 공급)
- 프리미엄 딜러 프로그램 (전용 물류 지원)

[장기]
- 해외 수입 대행 (해외 대리점 → 한국 소비자)
- B2B 마켓 (대리점 간 도매 거래)
- 데이터 기반 시장 인사이트 리포트 (브랜드별/카테고리별 할인 트렌드)
```
