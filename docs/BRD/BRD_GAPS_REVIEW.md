## 1. 사용자 페르소나 및 시장 분석

### 🔴 CRITICAL: 사용자 획득 전략 미정의 (Item #1)
**문제:** 각 페르소나별 DAU/MAU 목표는 있지만, "처음 10,000명을 어떻게 확보할 것인가"에 대한 구체적 Go-to-Market 전략이 전무합니다.
- Phase 1에서 누적 10만 명 설치 목표 vs 실제 고객 확보 방식 불명확
- 기존 WB3 사용자 마이그레이션 비율(몇 %?) 미정의
- 페르소나 1(탄소 감축형): 앱스토어 검색 유입만 가정? → 유료 광고 전략 없음
- 페르소나 2(도전형 러너): Strava 사용자 크로스 홍보? → 구체적 파트너십 전략 부족
- 크루 리더(페르소나 3): 기존 카톡/밴드 크루 500개를 얼마나 전환할 예정인가?

**CEO 확인 필요 사항:**
1. Phase 1 앱 출시 후 첫 90일 내 10,000명 확보를 위한 마케팅 예산 및 채널
2. 기존 WB3 회원 대 신규 사용자 목표 비율 (예: 30% 기존 / 70% 신규)
3. 각 페르소나별 초기 고객 확보 가장 가능성 높은 3가지 채널

**권장 액션:**
- Go-to-Market 전문 문서 작성 필요 (별도 3-5페이지 문서)
- 각 채널별 CAC(고객 획득 비용) 및 전환율 가정값 설정

### 🔴 CRITICAL: 페르소나 1 행동 변화 모멘트 불명확 (Item #2)
**문제:** 탄소 감축형 통근자(페르소나 1)가 "매일 아침 통근을 기록"하려는 강한 동기가 명확하지 않습니다.
- 기존 앱들(Strava, NRC)은 "운동한 기록을 남기고 친구와 공유"가 자명한 동기
- 우리의 경우 "탄소 계산 + SSP"가 진정한 동기인가?
- 통근자는 운동선수 아님 → 랭킹/경쟁 동기 약함
- 월 500~1,000 SSP = 약 5,000~10,000원 가치 vs 통근 번거로움 (3분의 앱 켜기)

**실제 문제:** 페르소나 1이 주말 크루 활동까지 전환되는 시나리오가 약해 보임.

**CEO 확인 필요 사항:**
1. 통근자가 앱을 매일 켤 유일한 이유가 SSP 자동 적립인가? 다른 동기는?
2. SSP 가치가 월 5천원일 때, 경쟁사(Strava 무료)대비 차별화 포인트는?
3. 페르소나 1에서 페르소나 2/3로 전환되는 비율 목표는? (기대값 vs 현실)

**권장 액션:**
- 페르소나 1 대상 심층 인터뷰 (기존 자전거 통근자 10명) 실시
- 월 SSP 가치를 점진적으로 조정할 시나리오 플래닝 (Phase 1 저가/Phase 2 정상가)

### 🟡 IMPORTANT: 페르소나별 성별/연령 세분화 부족 (Item #3)

1. Phase 1~3에서 타겟할 가장 "코어" 페르소나는 무엇인가? (단 1개 선택)
2. 여성 사용자의 초기 타겟 비율은?

**권장 액션:**

1. 경쟁사 한국 활성 사용자 수를 어디서 확인했는가? (공식 보도자료/증권사 리포트 링크 필요)

**권장 액션:**
- 경쟁 분석 섹션에 출처(URL/보도자료/증권사 리포트) 모두 명시

## 2. 수익 모델 및 재무

### 🔴 CRITICAL: Phase 1 손익분기 정의 불명확 (Item #6)
**문제:** 문서에서 "Phase 1은 수익 창출 없이 투자·개발 기간"이라고만 명시되어 있으나, 손익분기 달성 조건이 없습니다.
- Phase 1 운영비: 월 146만원 × 12개월 = 약 1,752만원
- 이는 어디서 나오나? (CEO 개인 자금? 투자금?)
- 광고 수익 시뮬레이션이 있지만(월 76만원 @ 10,000 MAU), 이를 달성할 확실성은?

**CEO 확인 필요 사항:**
1. Phase 1 동안 월 146만원 운영비를 누가 부담하나?
2. 광고 수익(AdMob) 월 76만원 달성이 "확실한가, 가정인가?"
3. 만약 광고 수익이 절반이면(월 38만원), 추가 자금 조달 계획은?

**권장 액션:**
- Phase 1 손익분기 분석: "MAU 몇 명 이상이면 자급자족 가능한가?"
  - AdMob 수익(월) = MAU × DAU비율(30%) × 광고 시청률(10%) × CPM(₩10,000)
  - 이 수식으로 손익분기 MAU 산정

### 🔴 CRITICAL: SSP 포인트 정책(Plan C) 실행 가능성 검증 부족 (Item #7)
**문제:** "통합 SSP + 소스 태깅"이 좋은 아이디어지만, 실행 후 발생할 수 있는 문제들이 미검토됨:

1. **사용자 혼란:** 사용자가 SSP 화면에서 "탄소 SSP 2,300 + 교환 가능 1,200"을 봤을 때, "왜 일부는 못 쓰나?"라고 문의할 것.
   - CS 비용 증가 예상

2. **기술 구현 복잡도:**
   - 모든 SSP 거래 시 source tag 추적 필요
   - 사용자가 AD SSP 1,000 + CARBON SSP 500을 섞어 사용 시 어느 것부터 차감?
   - 차감 로직(FIFO? LIFO? 혼합비율?)이 명확하지 않음

3. **정책 변경 리스크:**
   - Phase 4에서 CARBON SSP를 토큰으로 전환하기로 했지만, 그 전환율은?
   - "1 CARBON SSP = 1 토큰?"이면 너무 저가치
   - 토큰 가격 변동 시 사용자 분쟁 예상

**CEO 확인 필요 사항:**
1. SSP 태그 차감 순서 알고리즘은 무엇인가? (구체적 명세 필요)
2. CARBON SSP → 토큰 전환율은 Phase 4에서 어떻게 결정하나?
3. 사용자가 "왜 탄소 SSP는 기프티콘 못 쓰나?"라고 할 때 설득할 논리는?

**권장 액션:**
- SSP 사용처 UI/UX 와이어프레임 작성 (실제 화면 확인 필수)
- 토큰 전환율 메커니즘 문서 작성 (최소 Phase 2 후반까지)

### 🔴 CRITICAL: 깜깜이방 손익분기 GMV 설정이 너무 보수적 (Item #8)
**문제:** 손익분기 GMV가 "월 4,000만원 (약 80건, 평균 단가 50만원)"인데, 이는 너무 낮은 설정입니다.

- 평균 단가가 50만원이면, 딜러 마진이 매우 얇은 저가 상품(의류, 소모품) 위주 판매를 의미
- 문서에서 언급한 "완성차 300만원"을 한 달에 6~8대 파는 것만으로도 손익분기 가능하나, 이는 명시되지 않음
- 딜러 씨드 10개 확보 시 "딜러당 월 2~3건"이면 매우 낮은 회전율

**CEO 확인 필요 사항:**
1. 최초 30개 딜러를 어디서 확보할 예정? (WB3 기존 파트너? 새로운 채널?)
2. 각 딜러의 "합리적인 월간 판매 건수"는 몇 건인가? (2~3건은 매우 낮음)
3. Phase 3 런칭 시점에 딜러 씨드 몇 개까지 확보 완료되어야 "실패가 아닌가?"

**권장 액션:**
- 딜러 타입별(자전거 전문점 / 러닝 전문점 / 중규모 체인)로 월간 판매 건수 목표 설정
- "최소 성공 기준: 딜러 20개, 월 GMV 5,000만원" 등으로 명시

## 3. 기술 및 구현 전략

### 🔴 CRITICAL: 3Way Sensor 정확도 목표 vs 현실 (Item #12)
**문제:** 3Way Sensor의 핵심인 "자동 교통수단 감지" 정확도 목표가 명확하지만, 달성 가능성이 의문입니다.

문서: "3Way Sensor 자동 감지: 버스 90%, 지하철 95%, 자전거/도보 95%"

→ 현실 체크:
- **버스 감지 90%:** Wi-Fi SSID 패턴으로 가능한가?
  - 버스마다 Wi-Fi SSID가 다름 (예: 버스회사별 "WiBUS_12345", "GBus_456" 등)
  - 모든 버스 회사의 SSID를 DB화할 수 있나?
  - 시골 지역 버스는 Wi-Fi 없음 → GPS 속도로 감지? (정확도 떨어짐)

- **지하철 95%:** 가능할 수 있음 (지하철역 Wi-Fi는 통일)

- **자전거/도보 95%:** GPS 속도만으로? (보행자도 자전거도 속도 범위 겹침)

**실제 문제:** Phase 1에서 정확도 92% 달성이 KPI인데, "실패 시 어떻게 하나?"

**CEO 확인 필요 사항:**
1. 3Way Sensor의 기술 검증이 완료되었나? (개념 증명/PoC?)
2. 실제 통근자 50명 대상 2주 테스트는 언제 할 예정?
3. 정확도 92% 달성 불가 시 대체 전략은? (정확도를 내릴 것인가? 기능을 축소할 것인가?)

**권장 액션:**
- Phase 1 시작 전에 기술 PoC 완료 필수
