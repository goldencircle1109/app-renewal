# 제로페이(ZeroPay) 연동 심층 리서치

**작성일: 2026년 3월 25일**
**목적: 라이트브라더스 SSP 포인트 → 제로페이 전환/결제 연동 가능성 조사**

---

## 1. 시스템 아키텍처

### 1.1 전체 구조

```
[소비자 앱] → QR 스캔 → [KEPF/KFTC 중계] → [소비자 계좌 출금] → [가맹점 계좌 입금]
```

### 1.2 핵심 기관

| 기관 | 역할 | URL |
|------|------|-----|
| **한국간편결제진흥원(KEPF)** | 제로페이 플랫폼 운영, 가맹점 관리, QR 규격 | kepf.or.kr |
| **금융결제원(KFTC)** | 은행 간 결제망 운영 | kftc.or.kr |
| **KFTCVAN / KFTCPAY** | PG/VAN 서비스, 온라인 결제 중계 | kftcvan.or.kr / pg.kftcvan.or.kr |
| **비즈플레이(비플페이)** | 결제 앱 운영, 기업제로페이 | bizplaypay.co.kr |

### 1.3 QR 결제 흐름

**MPM (가맹점 비치 QR):**
1. 소비자 앱에서 QR 스캔
2. 결제 금액 입력 → 본인 계좌에서 즉시 출금
3. KFTC 금융공동망 → 가맹점 계좌 실시간 입금

**CPM (소비자 제시 QR):**
1. 소비자 앱에서 QR/바코드 생성
2. 가맹점 POS/스캐너로 스캔
3. POS → KFTCVAN → KFTC → 은행

### 1.4 수수료 구조

| 연매출 | 수수료 |
|--------|--------|
| 8억 이하 (소상공인) | **0%** |
| 8~12억 | 0.3% |
| 12억 초과 | 0.5% |

## 2. 연동 경로 비교

| 경로 | 방법 | 난이도 | 기간 | 라브 적합도 |
|------|------|--------|------|-----------|
| **A: KEPF 직접 참여** | 결제앱 참여사 등록 | 매우 높음 | 6~12개월 | ★ (카카오페이급만) |
| **B: KFTCPAY/PG 경유** | 온라인 결제에 제로페이 추가 | 보통 | 2~4개월 | ★★★ |
| **C: 기업제로페이 바우처** | PIN 발행 → 비플페이 사용 | **낮음** | **1~2주** | **★★★★★** |

---

## 3. 기업제로페이 바우처 (권장 경로)

### 3.1 개요

| 항목 | 상세 |
|------|------|
| 공식 사이트 | bizzeropay.com |
| 가이드 | bizzeropay.gitbook.io/bizzeropay |
| 바우처 가이드 | bizzeropay.gitbook.io/voucher |
| 사용자 가이드 | bizzeropay.gitbook.io/guide |

### 3.2 플로우

```
1. bizzeropay.com에 수요기업 가입
     ↓
2. 사용자 SSP → 제로페이 바우처 교환 요청
     ↓
3. 라브 서버: SSP 차감 + 바우처 PIN 발행 요청
     ↓
4. 8자리 PIN → 사용자에게 전달 (앱 내 또는 SMS)
     ↓
5. 사용자: 비플제로페이 앱에서 PIN 등록
     ↓
6. 전국 제로페이 가맹점에서 QR 결제
```

### 3.3 주요 기능

| 기능 | 설명 |
|------|------|
| 경비 지급 | 직원에게 출장비/업무비 모바일 지급 |
| 비대면 바우처 | 정부사업 수행 (중기부 바우처 등) |
| 지역상품권 | 지자체 협력 지역사랑상품권 연계 |

### 3.4 한계

- 사용자가 **비플제로페이 앱 별도 설치** 필요 → UX 마찰
- 라브 앱 내 직접 QR 결제는 경로 A(KEPF 참여)만 가능
- 공개 REST API 미확인 → 영업팀 직접 문의 필요

## 4. KFTCPAY 경유 (온라인 결제용)

### 4.1 서비스 신청

- URL: pg.kftcvan.or.kr/application/application
- 지원 결제: 신용카드 8개사, 계좌이체, 가상계좌, 휴대폰, **제로페이**

### 4.2 용도

- 앱 내 쇼핑몰에서 **제로페이 결제수단 추가**
- 사용자가 "제로페이" 선택 → 제로페이 결제창 이동 → 비플페이 앱으로 처리

## 5. 카카오페이–제로페이 연동 사례

### 5.1 연동 정보

| 항목 | 내용 |
|------|------|
| 시작일 | 2024년 4월 17일 |
| 가맹점 | 전국 **110만 QR 가맹점** (총 183만 중) |
| 방식 | 카카오페이가 KEPF에 **결제앱 참여사**로 공식 등록 |
| 결제수단 | 카카오페이머니, 포인트, 상품권 |

### 5.2 네이버페이·토스가 미연동인 이유

- 네이버페이: 자체 오프라인 결제 인프라 보유, 온라인 커머스 중심 전략
- 토스: 자체 토스페이 생태계 구축 집중, NFC 기반 방향
- 공통: 제로페이 수수료가 너무 낮아 수익성 확보 어려움

## 6. 금융결제원 오픈뱅킹 API

### 6.1 주요 API

| API | 설명 |
|-----|------|
| 잔액조회 | 사용자 계좌 잔액 확인 |
| 거래내역 조회 | 입출금 내역 |
| 출금이체 | 사용자 계좌 → 이용기관 계좌 |
| 입금이체 | 이용기관 → 제3자 계좌 |
| 사용자인증 | OAuth 2.0 계좌 등록 |

### 6.2 핀테크 서비스 등록 절차

1. openapi.kftc.or.kr에서 이용신청
2. 이용적합성 심사
3. 테스트베드 개발 (developers.open-platform.or.kr)
4. 금융보안원 보안점검
5. 이용계약 체결 → 실서비스

### 6.3 제로페이와의 관계

- 오픈뱅킹 출금이체로 "유사한 계좌이체 결제" 구현 가능
- 단, KEPF의 공식 제로페이 가맹점 QR 인프라와는 **별개**
- 공식 제로페이 연동은 KEPF 직접 계약 필요

## 8. 법적 요건

| 역할 | 전자금융업 등록 |
|------|---------------|
| 제로페이 이용 (바우처) | **불필요** |
| 결제앱 운영 (KEPF 참여) | **필요** (전자자금이체업) |
| PG 경유 결제 연동 | **불필요** (PG사가 보유) |
| PG사 직접 운영 | **필요** (자본금 3~10억) |

## 9. 라이트브라더스 권장 전략

### 1단계 (1~2개월): 기업제로페이 바우처

- bizzeropay.com 수요기업 가입
- SSP → 제로페이 바우처 PIN 발행
- 사용자가 비플페이 앱에서 등록 후 사용
- 금융업 등록 불필요

### 2단계 (장기): KEPF 결제앱 참여

- 라브 앱 내 QR 직접 결제 구현
- KEPF 참여사 등록 + 전자금융업 등록 검토
- 또는 전자금융업 등록된 파트너사(은행/핀테크)와 제휴
