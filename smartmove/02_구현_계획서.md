# SmartMove Activity Sensor 구현 계획서

> 작성일: 2026-03-19
> 프로젝트: Wright Brothers 앱 - 회원 활동 자동 인식 기능

---

## 1. 기능 개요

회원의 스마트폰 센서를 활용하여 **걷기, 뛰기, 자전거타기** 3가지 활동을 자동으로 인식하고, 활동 데이터를 기록하는 기능입니다.

### 핵심 목표
- 회원이 별도 조작 없이 자동으로 활동 인식
- 배터리 소모 최소화
- 실시간 + 히스토리 데이터 제공

---

## 2. 권장 구현 전략: 3단계 점진적 접근

### Phase 1: MVP - 플랫폼 내장 API 활용 (2~3주)

> **목표**: 최소한의 개발로 빠르게 동작하는 프로토타입 완성

#### 구현 내용
```
[Android] Google Activity Recognition Transition API
    → WALKING, RUNNING, ON_BICYCLE 감지
    → 신뢰도(confidence) 75% 이상만 유효 처리

[iOS] CMMotionActivityManager
    → walking, running, cycling 감지
    → confidence가 .medium 이상만 유효 처리
```

#### 주요 작업 항목

| 순서 | 작업 | 예상 기간 | 담당 |
|------|------|----------|------|
| 1 | 활동 인식 서비스 모듈 설계 | 2일 | - |
| 2 | Android Activity Recognition 연동 | 3일 | - |
| 3 | iOS Core Motion 연동 | 3일 | - |
| 4 | 활동 데이터 로컬 저장 구조 설계 | 2일 | - |
| 5 | 서버 API 설계 (활동 데이터 전송) | 2일 | - |
| 6 | 기본 UI (현재 활동 표시 + 오늘 기록) | 3일 | - |
| 7 | 테스트 및 QA | 3일 | - |

#### 기대 성과
- 정확도: 80~90%
- 배터리 추가 소모: 거의 없음
- 빠르게 사용자 피드백 수집 가능

---

### Phase 2: 정확도 향상 - GPS 보조 센서 융합 (2~3주)

> **목표**: 자전거 인식 정확도를 높이고, 활동 전환을 부드럽게 처리

#### 구현 내용
```
[GPS 속도 보조]
    → 활동 인식 결과 + GPS 속도를 교차 검증
    → 자전거: 신뢰도 낮을 때 GPS 속도 8~30km/h이면 자전거로 보정
    → GPS는 활동 감지 시에만 활성화 (상시 ON 금지 → 배터리 보호)

[활동 전환 스무딩]
    → 5초 윈도우 내 가장 빈번한 활동으로 확정
    → 급격한 전환 방지 (걷기→자전거→걷기 같은 노이즈 제거)

[걸음수 연동]
    → CMPedometer (iOS) / TYPE_STEP_COUNTER (Android)
    → 걷기/뛰기 시 걸음수, 이동 거리 추가 기록
```

#### 주요 작업 항목

| 순서 | 작업 | 예상 기간 |
|------|------|----------|
| 1 | GPS 속도 측정 모듈 (배터리 최적화) | 3일 |
| 2 | 센서 융합 로직 (API 결과 + GPS 보정) | 3일 |
| 3 | 활동 전환 스무딩 알고리즘 | 2일 |
| 4 | 걸음수/이동거리 연동 | 2일 |
| 5 | 활동 히스토리 상세 UI | 3일 |
| 6 | 서버 통계 API | 2일 |
| 7 | 테스트 및 QA | 3일 |

#### 기대 성과
- 정확도: 90~95%
- 자전거 인식 크게 개선
- 상세한 활동 데이터 (걸음수, 거리, 속도)

---

### Phase 3: 고도화 - 커스텀 ML 모델 (선택사항, 1~2개월)

> **목표**: 자체 데이터로 학습한 ML 모델로 최고 정확도 달성

#### 구현 내용 (필요 시에만)
```
[데이터 수집]
    → Phase 1~2에서 수집된 실제 사용자 데이터 활용
    → 사용자에게 "지금 뭐 하고 있어요?" 확인 팝업으로 라벨링

[모델 학습]
    → LSTM 또는 CNN 기반 경량 모델
    → 입력: 가속도계 3축 + 자이로스코프 3축 (2초 윈도우)
    → 출력: 걷기/뛰기/자전거/정지 4개 클래스

[온디바이스 배포]
    → Android: TensorFlow Lite (모델 크기 ~50KB 이하)
    → iOS: Core ML
    → 앱 업데이트 시 모델 동봉 또는 서버에서 다운로드
```

#### 기대 성과
- 정확도: 95~98%
- 폰 위치(주머니/손/가방)에 독립적인 인식
- 우리 회원의 패턴에 최적화된 모델

### 3-2. 데이터 흐름

```
1. 백그라운드 서비스 시작
   └→ Android: ActivityRecognitionClient 등록
   └→ iOS: CMMotionActivityManager.startActivityUpdates()

2. OS에서 활동 변화 감지
   └→ 콜백으로 활동 유형 + 신뢰도 수신

3. 신뢰도 검증
   └→ Android: confidence >= 75
   └→ iOS: confidence >= .medium

4. (Phase 2) GPS 교차 검증
   └→ 자전거 의심 시 GPS 속도 확인

5. 스무딩 처리
   └→ 5초 윈도우 내 최빈 활동으로 확정

6. 로컬 저장
   └→ { 활동유형, 시작시간, 종료시간, 신뢰도 }

7. 서버 동기화
   └→ Wi-Fi 환경에서 배치 전송 (또는 실시간)
```

### 3-3. 데이터 모델

```sql
-- 활동 기록 테이블
CREATE TABLE activity_records (
    id              INTEGER PRIMARY KEY,
    user_id         TEXT NOT NULL,
    activity_type   TEXT NOT NULL,    -- 'WALKING' | 'RUNNING' | 'CYCLING'
    started_at      DATETIME NOT NULL,
    ended_at        DATETIME,
    duration_sec    INTEGER,          -- 활동 지속 시간 (초)
    confidence      REAL,             -- 평균 신뢰도 (0.0~1.0)
    steps           INTEGER,          -- 걸음수 (걷기/뛰기만)
    distance_m      REAL,             -- 이동 거리 (미터)
    avg_speed_kmh   REAL,             -- 평균 속도 (km/h)
    calories        REAL,             -- 소모 칼로리 (추정)
    synced          INTEGER DEFAULT 0, -- 서버 동기화 여부
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 일별 요약 테이블
CREATE TABLE daily_summary (
    id              INTEGER PRIMARY KEY,
    user_id         TEXT NOT NULL,
    date            DATE NOT NULL,
    walk_minutes    INTEGER DEFAULT 0,
    run_minutes     INTEGER DEFAULT 0,
    cycle_minutes   INTEGER DEFAULT 0,
    total_steps     INTEGER DEFAULT 0,
    total_distance  REAL DEFAULT 0,
    total_calories  REAL DEFAULT 0,
    UNIQUE(user_id, date)
);
```

## 7. 칼로리 계산 공식

활동별 MET(Metabolic Equivalent of Task) 값을 기반으로 추정합니다.

```
칼로리(kcal) = MET × 체중(kg) × 시간(hour)

MET 값:
- 걷기 (보통 속도 4km/h): 3.0
- 걷기 (빠른 속도 6km/h): 3.8
- 뛰기 (조깅 8km/h): 8.0
- 뛰기 (달리기 12km/h): 11.5
- 자전거 (보통 15km/h): 6.0
- 자전거 (빠름 25km/h): 10.0

예시: 70kg 회원이 30분 조깅 시
= 8.0 × 70 × 0.5 = 280 kcal
```
