# AI 챗봇 신규 기능 기획서

> 작성일: 2026-03-28
> 근거: 채널톡 CS 데이터 890건 분석 (2021.03 ~ 2026.03, 11개 시기 샘플링)
> 목적: 앱 리뉴얼 시 채널톡 → AI 챗봇 전환

---

## 1. 왜 AI 챗봇인가 — 데이터 근거

### 현재 CS 현황 (채널톡 기준)

| 지표 | 수치 |
|------|------|
| 총 대화 | 34,088건 (2020.10 ~ 현재) |
| 월 평균 | ~520건 |
| 상담사 응답률 | 74~99% (시기별 편차 큼) |
| 대화당 평균 메시지 | 7~14건 |

### 자동화 가능한 문의 비율

890건 분석 결과, **전체 문의의 약 70%가 패턴화된 반복 문의**임:

| 카테고리 | 비율 | AI 자동 처리 가능 여부 |
|---------|------|---------------------|
| 매입/위탁/중고거래 | 28% | ✅ 서비스 종료 안내 + 대안 연결 |
| 수리/AS/비파괴검사 | 19% | ✅ 예약/가격/절차 안내 |
| 상품문의/재고 | 13% | ✅ DB 연동 실시간 조회 |
| 배송/수령 | 10% | ✅ 주문 조회 연동 |
| **자동화 가능 소계** | **70%** | |
| 결제/정산 | 8% | ⚠️ 부분 자동화 (안내만) |
| 기타/복합 | 22% | ❌ 사람 상담 필요 |

### 예상 효과

- 상담사 업무량 **70% 감소**
- 24시간 즉시 응답 (현재 평균 응답 지연 있음)
- 일관된 답변 품질

## 2. AI 챗봇 핵심 기능 설계

### 2-1. 자동 문의 분류 엔진 (Intent Classification)

890건 분석에서 도출한 **10개 카테고리**로 자동 분류:

```
사용자 메시지 입력
    ↓
[Intent 분류 엔진]
    ↓
┌─────────────────────────────────────────┐
│ 1. 매입/위탁/중고거래  → 서비스 안내 봇    │
│ 2. 수리/AS/비파괴검사  → 예약 봇          │
│ 3. 상품문의/재고      → 재고 조회 봇      │
│ 4. 배송/수령         → 배송 조회 봇      │
│ 5. 결제/정산         → 결제 안내 봇      │
│ 6. 환불/취소         → 환불 처리 봇      │
│ 7. 렌탈            → 렌탈 안내 봇      │
│ 8. 계정/인증         → 계정 도움 봇      │
│ 9. 예약/방문         → 예약 봇          │
│ 10. 투어/이벤트      → 이벤트 안내 봇    │
│ 11. 분류 불가        → 사람 상담 연결    │
└─────────────────────────────────────────┘
```

**구현 방식:**
- LLM 기반 분류 (Claude Haiku) — 자연어 이해, 복합 문의 처리
- 키워드 폴백 — LLM 장애 시 규칙 기반 분류
- 대화당 분류 비용: ~$0.001 (Haiku 기준)

---

### 2-2. 매입/위탁/중고거래 봇 (문의 비율 1위: 28%)

**배경**: 서비스가 종료되었으나 문의가 가장 많은 카테고리. 2021년부터 현재까지 일관되게 20~38%를 차지.

**실제 고객 문의 패턴:**
- "자전거 매입 가능한가요?"
- "위탁판매 하고 싶은데요"
- "중고 자전거 팔고 싶습니다"
- "바꾸기 서비스 이용하고 싶어요"

**봇 동작:**

```
[고객] "자전거 팔고 싶은데요"

[AI 봇]
안녕하세요! 자전거 판매 관련 문의를 주셨군요.

라이트브라더스의 직접 매입/위탁판매 서비스는 현재 운영하지 않고 있습니다.

대신 이용하실 수 있는 방법을 안내드릴게요:

🔄 라이트브라더스 앱 내 중고거래
  → 개인 간 직접 거래 (앱에서 바로 등록 가능)

🚲 외부 중고 플랫폼
  → 자전거 전문: 바이크매니아, 빈티지로드
  → 종합: 중고나라, 번개장터

판매 등록 방법을 안내해 드릴까요?
```

**핵심 로직:**
- 서비스 종료 사실 안내
- 대안 경로 자동 연결 (앱 내 중고거래 등록 페이지 딥링크)
- 추가 질문 시 사람 상담 전환

---

### 2-3. 비파괴 검사/수리/AS 봇 (문의 비율 2위: 19%)

**배경**: 2021년 5%에서 2025년 19%로 꾸준히 성장. 라이트브라더스의 차별화 서비스.

**실제 고객 문의 패턴:**
- "비파괴 검사 비용이 얼마인가요?"
- "비파괴 검사 예약하고 싶습니다"
- "프레임 크랙 확인이 가능한가요?"
- "정품 확인을 위해 검사를 받고 싶어요"
- "지방인데 방문 가능한가요?"
- "검사 결과는 언제 나오나요?"

**봇 동작 — 정보 안내 모드:**

```
[고객] "비파괴 검사 비용이 얼마인가요?"

[AI 봇]
비파괴 검사 서비스 안내드리겠습니다.

📋 검사 종류 및 비용
  • 기본 프레임 검사: {price_basic}원
  • 정밀 검사 (정품 확인 포함): {price_premium}원
  • 사고 프레임 검사: {price_accident}원

⏱️ 소요 시간: 방문 당일 약 {duration}시간

📍 검사 장소: 서울 성동구 라이트브라더스

검사 예약을 도와드릴까요?
```

**봇 동작 — 예약 모드:**

```
[고객] "예약하고 싶어요"

[AI 봇]
검사 예약을 도와드리겠습니다.

아래 정보를 입력해 주세요:
1. 고객명:
2. 연락처:
3. 자전거 모델:
4. 자전거 연식:
5. 검사 사유: (정품 확인 / 사고 검사 / 기타)
6. 희망 방문일:

[양식 자동 생성 → DB 저장 → 상담사 알림]
```

**핵심 로직:**
- 가격/절차 정보는 DB에서 실시간 조회 (관리자가 수정 가능)
- 장비 상태 (가동/점검 중) 자동 표시
- 예약 양식 → DB 저장 후 상담사에게 알림
- 방문 예약 캘린더 연동 (빈 슬롯 자동 표시)

### 2-4. 상품문의/재고 봇 (문의 비율 3위: 13%)

**배경**: 2021년 43%에서 13%로 감소했으나 여전히 주요 문의. "재고 있나요?" 질문이 대부분이며, 상담사 응답이 "재고가 없습니다"로 끝나는 경우가 많음.

**실제 고객 문의 패턴:**
- "BMC 팀머신 SLR SEVEN 51사이즈 재고 있나요?"
- "엔비 이너드라이브 시마노바디만 구매 가능할까요?"
- "바도 5.0 있나요?"
- "트렉 마돈 가격 알려주세요"

**봇 동작:**

```
[고객] "BMC 팀머신 51사이즈 재고 있나요?"

[AI 봇]
BMC 팀머신 재고를 확인해 드리겠습니다.

🔍 검색 결과:
  ❌ BMC 팀머신 SLR SEVEN 51 — 현재 재고 없음

💡 대안 추천:
  ✅ BMC 팀머신 SLR FIVE 51 — 재고 2대 (₩3,500,000)
  ✅ BMC 팀머신 SLR SEVEN 54 — 재고 1대

📩 입고 알림을 등록하시겠어요?
  → 원하시는 모델이 입고되면 푸시 알림으로 알려드립니다.
```

**핵심 로직:**
- **상품 DB 실시간 연동**: 재고 수량, 가격, 사이즈 조회
- **유사 상품 자동 추천**: 같은 브랜드/카테고리/사이즈에서 재고 있는 상품 표시
- **입고 알림 등록**: 관심 상품 재입고 시 푸시 알림 (신규 기능)
- **상품 페이지 딥링크**: 앱 내 상품 상세 페이지로 바로 이동

---

### 2-5. 배송/수령 봇 (문의 비율 4위: 10%)

**배경**: 2022년 38%에서 10%로 크게 감소. 배송 프로세스가 개선된 것으로 보이나, 주소 변경/수령 방법 문의는 여전히 존재.

**실제 고객 문의 패턴:**
- "배송 언제 오나요?"
- "주소 변경해야 합니다"
- "방문수령으로 변경 가능한가요?"
- "주말에 방문수령 가능한가요?"

**봇 동작:**

```
[고객] "주문한 자전거 배송 언제 오나요?"

[AI 봇]
주문 배송 현황을 조회해 드리겠습니다.

📦 주문번호: WB-2026-XXXXX
  상품: 트렉 마돈 SL 6
  상태: 출고 완료 (03/25)
  운송장: CJ대한통운 123456789
  예상 도착: 03/28 (금)

🔗 [실시간 배송 추적하기]

추가 도움이 필요하신가요?
  • 배송 주소 변경
  • 방문수령으로 변경
  • 상담사 연결
```

**핵심 로직:**
- **주문 DB 연동**: 회원 ID로 주문 상태 자동 조회
- **택배사 API 연동**: 운송장 번호로 실시간 배송 추적
- **주소 변경**: 출고 전이면 자동 처리, 출고 후면 상담사 연결
- **방문수령 안내**: 영업시간, 예약 필요 여부 자동 안내

---

### 2-6. 결제/정산 봇 (문의 비율 5위: 8%)

**배경**: 시기별로 3~11%로 편차가 큼. 렌탈(이니렌탈) 도입 이후 결제 관련 문의가 증가.

**실제 고객 문의 패턴:**
- "카드 한도 초과로 결제가 안 돼요"
- "렌탈 결제수단 변경 가능한가요?"
- "정산 언제 되나요?" (판매자)
- "결제 모듈이 웹에서 안 됩니다"

**봇 동작:**

```
[고객] "렌탈 결제수단 변경하고 싶어요"

[AI 봇]
렌탈 결제수단 변경은 이니렌탈을 통해 처리됩니다.

📞 이니렌탈 고객센터: 1800-1739
  • 영업시간: 평일 09:00~18:00
  • 본인 확인 후 변경 가능

필요한 정보:
  • 계약자 성함
  • 계약 번호 (앱 > 마이페이지 > 렌탈 관리에서 확인)

다른 도움이 필요하신가요?
```

**핵심 로직:**
- 렌탈 관련 → 이니렌탈 안내로 자동 라우팅
- 결제 오류 → 결제 시스템 상태 자동 체크 + 트러블슈팅 가이드
- 정산 문의 (판매자) → 정산 예정일 DB 조회
- 해결 불가 시 → 상담사 연결 (결제 이슈는 민감)

---

### 2-7. 렌탈 안내 봇 (문의 비율: 2~6%, 증가 추세)

**배경**: 2023년부터 등장, 점차 증가 중. 이니렌탈 연동으로 인한 문의.

**봇이 처리할 문의:**
- 렌탈 신청 방법/조건
- 렌탈 계약 조회
- 결제수단 변경 안내 (이니렌탈 연결)
- 중도 해지 절차
- 렌탈 잔여 기간 확인

---

### 2-8. 스마트 상담사 연결 (Escalation)

AI가 해결할 수 없는 경우의 상담사 전환 로직:

**자동 전환 트리거:**
1. 고객이 "사람과 상담하고 싶어요" 요청
2. AI가 3회 이상 동일 질문에 답변 실패
3. 감성 분석에서 부정(분노/좌절) 감지
4. 결제/환불 등 금전 관련 복잡한 이슈
5. 민원/불만 키워드 감지

**전환 시 상담사에게 전달할 정보:**
```
[자동 요약]
  • 고객: 홍길동 (ID: xxx, VIP 등급)
  • 문의 유형: 환불/취소
  • AI 대화 요약: "렌탈 자전거 계약 취소 요청. 이니렌탈 안내했으나 해결 안됨"
  • 감정 상태: 불만 (3회 문의)
  • 관련 주문: WB-2026-XXXXX
```

### 3-2. LLM 선택

| 용도 | 모델 | 이유 |
|------|------|------|
| Intent 분류 | Claude Haiku 4.5 | 빠르고 저렴 ($1/MTok) |
| 응답 생성 | Claude Haiku 4.5 | 일반 안내는 가벼운 모델로 충분 |
| 복잡한 상담 | Claude Sonnet 4.5 | 감성 이해, 복합 문의 |

### 3-3. 예상 비용 (월 520건 기준)

| 항목 | 비용 |
|------|------|
| Intent 분류 (Haiku) | ~$0.5 |
| 응답 생성 (Haiku) | ~$2.0 |
| 복잡 상담 (Sonnet, ~30%) | ~$5.0 |
| **합계** | **~$7.5/월 (약 1만원)** |

채널톡 Growth 요금제(월 $96~) 대비 **약 90% 비용 절감**

## 4. 데이터 기반 우선순위

890건 분석에서 도출한 **자동화 대상 키워드 빈도**:

## 5. 시기별 트렌드에서 도출한 추가 기능

### 5-1. 매입/위탁 → 중고거래 전환 유도

- 2021~현재까지 20~38%로 가장 높은 문의
- 서비스 종료 안내에서 끝나지 않고, **앱 내 중고거래 등록으로 유도**
- 봇이 판매 등록 양식을 안내하고 사진 업로드까지 유도

### 5-2. 비파괴 검사 예약 시스템

- 2021년 5% → 2025년 19%로 꾸준히 성장
- 현재 문의 → 상담사 수동 처리 → 예약 확인 전화 (비효율)
- **봇에서 직접 예약 가능** (빈 슬롯 표시 → 선택 → 확인)

### 5-3. 렌탈 허브

- 2023년부터 등장, 점점 증가 추세 (0% → 6%)
- 이니렌탈 FAQ 자동 안내
- 렌탈 계약 조회 (이니렌탈 API 연동 가능 시)
- 렌탈 상품 추천

### 5-4. 배송 문의 자동화 (감소 추세지만 자동화 가치 높음)

- 38% → 10%로 감소했지만, 남은 문의는 "주소 변경", "방문수령 변경" 등 조치가 필요한 건
- 주문 API 연동으로 상태 조회 + 주소 변경 자동 처리

### 5-5. 응답률 안정화

- 현재 74~99%로 편차가 큼 (상담사 가용에 따라)
- AI 봇은 24시간 100% 응답 보장
- 상담사는 AI가 에스컬레이션한 30%에만 집중

## 6. AI 챗봇에만 있는 차별화 기능

### 6-1. 대화 맥락 기억

```
[3월 1일]
고객: "비파괴 검사 예약했는데 언제 가능한가요?"
봇: "현재 장비 이전 중이라 3월 중 재개 예정입니다. 재개되면 알림 드릴까요?"

[3월 20일]
고객: "그때 말한 비파괴 검사 예약 가능한가요?"
봇: "네! 이전에 문의하신 비파괴 검사, 현재 예약 가능합니다.
     이전 대화에서 말씀하신 피나렐로 프레임 검사로 예약 도와드릴까요?"
```

### 6-2. 선제적 안내 (Proactive Messaging)

주문/렌탈/예약 상태 변경 시 고객이 묻기 전에 먼저 알림:

```
[푸시 알림]
"홍길동님, 주문하신 트렉 마돈의 배송이 시작되었습니다!
 예상 도착: 3/28(금). 실시간 추적은 여기를 눌러주세요."
```

### 6-3. 이미지 인식 (추후)

- 자전거 사진 업로드 → 모델 자동 식별
- 파손 사진 → 수리 견적 자동 산출
- 중고 사진 → 시세 자동 조회

### 6-4. CS 데이터 자동 분석 대시보드

- 문의 유형별 실시간 분포
- 고객 감성 트렌드
- 반복 문의 패턴 자동 감지 → FAQ 업데이트 제안
- 상담사 개입 필요 건 알림

## 7. 구현 로드맵

### Phase 1: MVP (4주)

| 주차 | 작업 | 기능 |
|------|------|------|
| 1주 | 챗봇 UI + 기본 엔진 | 채팅 인터페이스, Intent 분류 |
| 2주 | 고정 안내 봇 3개 | 매입 안내, 비파괴 안내, 영업시간 |
| 3주 | DB 연동 봇 2개 | 재고 조회, 배송 조회 |
| 4주 | 상담사 연결 | 에스컬레이션, 대시보드 기본 |

**MVP로 자동화 가능한 문의: ~50%**

### Phase 2: 고도화 (4주)

| 주차 | 작업 | 기능 |
|------|------|------|
| 5주 | 비파괴 검사 예약 시스템 | 캘린더 연동, 예약 양식 |
| 6주 | 렌탈 안내 + 결제 안내 | 이니렌탈 FAQ, 결제 트러블슈팅 |
| 7주 | 대화 맥락 기억 | 이전 대화 참조, 개인화 |
| 8주 | 선제적 알림 | 배송/예약 상태 변경 푸시 |

**Phase 2 완료 시 자동화: ~70%**

### Phase 3: 지능화 (추후)

- 이미지 인식 (자전거 모델 식별, 파손 인식)
- CS 분석 대시보드
- 고객 이탈 예측 + 선제 대응
- A/B 테스트 (응답 방식 최적화)

## 8. 채널톡 vs AI 챗봇 비교

| DB 연동 | 불가 | **재고/주문/예약 연동** |
